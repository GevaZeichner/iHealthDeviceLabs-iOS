– setTimeWithDate:timezone:successBlock:errorBlock:

Set time

- (void)setTimeWithDate:(NSDate *)date timezone:(float)timezone successBlock:(BG5SSuccessBlock)successBlock errorBlock:(BG5SSDKErrorBlock)errorBlock

Parameters

date

the date you want to set should be an UTC time. valid range:Jan. 1,2000 - Dec. 31,2099. If date is nil, use [NSdate date] and system time zone
timezone

the timezone you want to set should be from -12.00 to 14.00; step length = 0.25
successBlock

the block means success
errorBlock

the block means fail. You can get error from this block

Discussion

Note: If date is out of reange, BG5SError_InputParametersError will be returned by block. If date and timezone are invalid , use input parameters; otherwise use [NSdate date] and system time zone;

– startMeasure:withSuccessBlock:errorBlock:

Start online measurement with specific measure mode.

- (void)startMeasure:(BGMeasureMode)measureMode withSuccessBlock:(BG5SSuccessBlock)successBlock errorBlock:(BG5SSDKErrorBlock)errorBlock

Parameters

measureMode

Blood or CTL
successBlock

the block means success
errorBlock

the block means fail. You can get error from this block

Discussion

Note: If you don’t call this method, BG5S use blood mode by default.

If CTL mode is sent, BG5S use CTL mode and BG5S can’t be set to use blood mode until next connection.
